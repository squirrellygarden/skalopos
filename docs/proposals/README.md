# Proposals

When a piece of work appears to require changing or adding a design pillar, write a short proposal here before implementing. Format is loose; keep it short. A typical proposal is one page.

## Template

```markdown
# Proposal: <short title>

## Problem
What does the existing design fail to do, or do badly? Be specific. Cite the affected pillar(s).

## Proposed change
The smallest possible change to the design that solves the problem.

## Alternatives
At least two other ways the problem could be solved, and why this one wins.

## Cost
What does this break? Existing pillars? Existing source files? Existing docs?

## Status
draft / accepted / rejected / superseded by <link>
```

## Why proposals exist

The eleven pillars are explicit commitments. Implementing against them is straightforward work. *Changing* them is a design decision that needs to be visible, debated, and recorded. Without that discipline, the design drifts; with it, every change is intentional.

Old proposals (rejected or superseded) stay in this directory. They're a record of "we already thought about this," which saves future-you from reopening the same question.
